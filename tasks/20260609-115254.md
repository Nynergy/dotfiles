# [isli] Read tasks once

- Created:  June 09, 2026
- Status:   CLOSED

Listing the tasks and opening a task both read in the tasks. Even though these
functions are technically mutually exclusive, I want to read in all the tasks in
a single place, preferably before we list or open tasks, but after we create a
new issue.

We can bake the status and ALL flag into the single task read loop, then listing
and opening tasks will just operate on the opaque "task list" array.
